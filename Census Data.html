Census Data

- Working with U.S. Census Data and TIGER/line® geography data

Problem: The employment rate has been fluctuating in 2020 more than ever compared to the previous years in the U.S because of the pandemic; as a researcher, I would like to know more data about the employment status in previous years, specifically in 2018 or 2019. As a current resident of North Carolina, the local and state data will be my focus using census data to observe status in recent years and using spatial analysis tools to find if there are any patterns.

Procedures:

Methods: (Software utilized: ArcMAP 10.7.1)

After downloading the two original datafiles, I found the information, which is the employment/population ratio, and edited the dataset in order to get a simpler representation. Also, the dataset that was downloaded from Tiger shapefile website has been examined to find the same field to add join with the employment dataset. Before applying the Add Join tool, I converted the excel file to table and created a new field within the employment and calculated the field based on GEOID information. When joining the two datasets, the field GEOID ,which is the five-digit number of information that represents each county, is the parameter of our focus. To present the results of employment rate that is only within North Carolina State Boundary, the attribute selection is performed based on the five-digit county code. Then to show the employment rate within NC, I then made some changes to the symbology and decided the range I would like to use, which used 5% increments of each range, and each has been changed to a graduate color scheme.


Results:

Figure.1 Display of the Distribution of Employment Rate in Each County of North Carolina in 2018

Reflections:

Difficulties Encountered: Firstly, when I noticed the two datasets both have 5-digit county information, I tried to Apply Add Join tool directly, but it did not work. There are some errors showing up, such as saying the column is not indexed. My solution was that after searching more information about what and how to index the column. After I understood the concept, then I was able to perform the join correctly.

Evaluation: The datasets which has the employment information has many columns and it’s very easy to get confused with what has been provided. In order to avoid the possible error that may occur when running the spatial analysis tools, I eliminated the columns from the datasets that not necessary for this practice. This could be helpful when dealing with other type of datasets as well.


Application & Reflection:

Analyzing census datasets can be applied to many other scenarios. For instance, if I would like to find out the distribution of the schools within a county the states of my interests, I am able to find the datasets that have schools' information and joining with the state boundary to show the results, which is very similar to this practice. The concept is to join a table with a shapefile based on the same attribute, and it can be performed when we have the appropriate datasets. For this specific scenario, the layer that has the school information will likely to have many schools sorted into different zip code, and if the zip code is presented in the county boundary layer, the field that matches will be the zip code. Then we will be able to find the distribution within each zip code and possibly perform further spatial and statistical analysis such as high/low values clusters within our study area.

Learning goals:

  1. Search for, retrieve, evaluate the suitability of, and integrate datasets for specific types of spatial analysis applications.

  2. Develop the ability to independently locate, filter, link, and map U.S. Census Bureau tabular data and TIGER/Line® shapefiles.

  3. Integrate and analyze data in various formats.

  4. Demonstrate ability to apply global and/or local pattern analysis to U.S. Census Data.
